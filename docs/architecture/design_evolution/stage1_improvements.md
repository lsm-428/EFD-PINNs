```python
⚠️ Stage1 局限性与使用建议：
    ========================================
    1. 降压过程简化：

       - 观察现象：同一电压下升压/降压路径不同
       - 物理本质：瞬态响应的时间尺度差异，非真实迟滞
       - Stage2 应学习：完整的 Navier-Stokes 动态，而非简单路径

    4. 二维到三维的简化：
       - Stage1: 仅预测 θ 和 η 两个标量
       - Stage2: 学习完整三维流场 (u, v, w, p, α)
       - 建议：Stage1 数据仅作为软约束/初值猜测
    ========================================
"""
```

- 在文件开头就明确告知使用者 Stage1 的局限性
- 提供具体的使用建议,指导 Stage2 训练策略

### 2. **修正"降压响应"的误导性注释** (第28-32行)

**修改前**:
```python
物理机制：
    降压 (V_from > V_to)：
        - 接触角：瞬间恢复到目标值（Young-Lippmann 是瞬态方程）
        - 开口率：表面张力驱动油墨铺展，一阶指数衰减 η(t) = η_0 × e^(-t/τ_recovery)
```

**修改后**:
```python
物理机制（Stage1 简化假设）：
    降压 (V_from > V_to)：
        - ⚠️ **Stage1 简化假设**：接触角假设快速恢复（非瞬间，但相对开口率恢复更快）
        - 实际物理：接触角由 Young-Lippmann 方程给出目标值，但实际恢复受黏性/惯性影响
        - 开口率：表面张力驱动油墨铺展，一阶指数衰减 η(t) = η_0 × e^(-t/τ_recovery)
        - **注意**：此简化可能高估降压速度，Stage2 PINN 应学习真实的三维流场动态
```

**改进理由**:
- **核心问题**: 原注释说"瞬间恢复",但实际输出显示 θ 从 67.5° → 119.9° 需要 50ms,并非瞬间
- **修正**: 强调这是"快速恢复"的**简化假设**,而非真实物理
- **数据支撑**: 实际运行结果显示,降压过程在 10-50ms 区间仍有明显变化

### 4. **修正"迟滞环"的物理描述** (第328-339行)

**修改前**:
```python
def demo_voltage_sweep(model: Stage1AnalyticalModel):
    """电压扫描演示（迟滞环）"""
    print("\n" + "=" * 70)
    print("5. 电压扫描 (0V → 30V → 0V 迟滞环)")
    print("=" * 70)
```

**修改后**:
```python
def demo_voltage_sweep(model: Stage1AnalyticalModel):
    """电压扫描演示（迟滞环）

    ⚠️ 注意：此"迟滞环"实际上是瞬态响应的路径依赖
        - 升压过程：欠阻尼振荡上升
        - 降压过程：指数衰减恢复
        - 同一电压下差异 ≠ 真实迟滞（是时间尺度不同）
    """
    print("\n" + "=" * 70)
    print("5. 电压扫描 (0V → 30V → 0V) - 路径依赖演示")
    print("=" * 70)
    print("⚠️ 注意：此差异是瞬态响应的路径依赖，非真实迟滞")
```

**改进理由**:
- **物理准确性**: "迟滞环"通常指材料本身的历史依赖性(如铁磁性)
- **Stage1 实际**: 是**路径依赖的瞬态响应**,即升压/降压的时间常数不同
- **数据验证**: 在 25V 下,升压路径 θ=85.1°/η=77.1%,降压路径 θ=120.0°/η=36.2%
  - 这种巨大差异是因为 `t_per_step=2ms`,降压阶段还没达到稳态
  - 如果 `t_per_step` 足够长,两条路径最终会收敛到同一稳态

## 📊 对 Stage2 训练的影响

### 修改前可能存在的问题:
1. **过度依赖 Stage1**: 如果使用者认为 Stage1 完全准确,可能在 Stage2 中设置过强的边界条件约束
2. **采样不足**: 在降压初期(t < 10ms)采样密度不够,导致 Stage2 学习不到真实动态
3. **权重失衡**: 可能给 Stage1 数据过高的权重,抑制了 PINN 学习三维物理

### 修改后的改进:
1. **明确假设**: 清楚标注 Stage1 的简化之处
2. **指导采样**: 建议在关键区域(降压初期)增加采样密度
3. **权重建议**: 建议 Stage1 数据作为**软约束**,允许 PINN 偏离 Stage1 学习真实物理
