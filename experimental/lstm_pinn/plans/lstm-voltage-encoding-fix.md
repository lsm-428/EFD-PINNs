# LSTM 电压编码修复计划

**创建时间**: 2026-03-13
**目标**: 修复 LSTM 等效电压输出错误，使模型预测符合物理规律
**验证方式**: 预测结果与训练数据目标一致

---

## 问题诊断

### 核心问题

**LSTM 输出的等效电压完全错误，导致 phi_net 收到错误输入**

### 实测数据

| 输入 | 预期 V_eff | 实际 V_eff | 预期 V_prev | 实际 V_prev |
|------|------------|------------|-------------|-------------|
| 0→30V, 5ms | 30V | 28.1V | 0V | **24.7V** ❌ |
| 0→30V, 10ms | 30V | 27.9V | 0V | **23.5V** ❌ |
| 0→30V, 20ms | 30V | **21.5V** ❌ | 0V | **13.0V** ❌ |
| 0→30V, 30ms | 30V | **18.8V** ❌ | 0V | 2.7V |

### 根因分析

1. **辅助损失覆盖不全**：只在单步场景应用，但单步场景输出也不对
2. **辅助损失权重过低**：`aux_weight = 0.5`，相对 data_loss 太小
3. **LSTM 学习了错误的映射**：把 t_since 混入了电压编码
4. **phi_net 收到错误输入**：无法正确预测 φ

### 连锁影响

```
LSTM 输出错误 → phi_net 输入错误 → φ 预测错误
     ↑
辅助损失权重过低 + 覆盖不全
```

---

## 解决方案对比

| 方案 | 描述 | 优点 | 缺点 | 推荐度 |
|------|------|------|------|--------|
| **A: 简化架构** | 直接使用输入电压，不通过 LSTM | 简单可靠，立即可用 | 失去 LSTM 编码能力 | ⭐⭐⭐⭐⭐ |
| **B: 增强辅助损失** | 增大权重，覆盖所有场景 | 保留 LSTM 灵活性 | 需要重新训练 | ⭐⭐⭐⭐ |
| **C: 两阶段训练** | 先训练 LSTM 输出，再训练 phi_net | 分离优化目标 | 复杂度高 | ⭐⭐⭐ |
| **D: 修改 LSTM 输入** | 移除 t_since，只输入 V_from, V_to | 避免混淆 | 损失时序信息 | ⭐⭐ |

---

## 推荐方案：A - 简化架构

### 设计

```python
# 当前架构（有问题）
voltage_seq → LSTM → (V_eff, V_prev_eff) → phi_net → φ

# 简化架构（推荐）
voltage_seq → 直接提取 → (V_eff, V_prev_eff) → phi_net → φ
```

### 实现

```python
# 在 forward() 中
# 不再使用 LSTM 输出的等效电压
# 直接使用输入序列的电压值

V_eff = voltage_seq[:, -1, 1]      # 最后一步的 V_to（归一化）
V_prev_eff = voltage_seq[:, 0, 0]  # 第一步的 V_from（归一化）
```

### 理由

1. **立即可用**：不需要重新训练
2. **语义正确**：V_eff = 目标电压，V_prev = 起始电压
3. **保留 LSTM**：LSTM 仍然用于编码时序信息（hidden state 传给 phi_net）
4. **简化调试**：问题范围缩小到 phi_net

## 风险与回退

### 风险

1. **phi_net 依赖了错误的 V_eff/V_prev_eff**：需要重新训练
2. **简化架构损失灵活性**：多步序列可能效果差

### 回退方案

如果方案 A 效果不佳：
1. 回退到方案 B（增强辅助损失）
2. 完全重新训练
3. 考虑端到端架构（不用 LSTM）
